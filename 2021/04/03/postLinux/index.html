入门概述

我们为什么学Linux

  • 在服务器端，在开发领域linux屹立不倒
  • 一些大的项目部署在服务器都是用的Linux
  • 开源社区Linux屹立不倒
  • 一位后端程序员必须掌握的技能

什么是Linux

Linux 是一套免费使用和自由传播的类 Unix 操作系统，是一个基于 POSIX（可移植操作系统接口） 和 UNIX 的多用户、多任务、支持多线程和多 CPU 的操作系统。

Linux 能运行主要的 UNIX 工具软件、应用程序和网络协议。它支持 32 位和 64 位硬件。Linux 继承了 Unix 以网络为核心的设计思想，是一个性能稳定的多用户网络操作系统。

Linux一切皆文件。

系统分区

  • 每4kb放成一个小隔断

  • /dev/后面放的都是硬件设备文件名

  • 给分区分配盘符成为挂载

  • 顺序 分区-格式化-给每个分区设文件名-写入挂载点（盘符） 分区既可以开始使用了

  • windows使用CDEF作为盘符，linux使用空目录名称作为盘符。理论上都可以作为盘符

  • 分区：把大硬盘变成小的逻辑分区

  • 格式化：写入文件系统

  • 分区设备与文件名：给每个分区定义设备文件名

  • 挂载：给每个分区分配挂载点（必须是空目录）

远程登录管理

个人用户选择Desktop安装 服务器使用Minimal（最小化）安装

root用户最高权限

ifconfig eth0 192.168.1.156 设置linux的ip地址

Linux注意事项

linux严格区分大小写

linux中所有的内容都是以文件的形式保存的

linux不靠扩展名区分文件类型 ，没有扩展名的规则 靠的是 文件权限

Linux也可以靠扩展名区分文件 如 压缩包 .gz .bz2 .tar.bz2 .tgz

二进制软件包 .rpm 网页文件 .html .php

脚本文件 .sh 配置文件 .conf

不写也可以 但是为了管理员方便 系统是可以自己识别

ls /etc/ 查看配置文件

sudo passwd root 修改root秘密 (在Ubuntu中)

linux所有的存储设备都必须挂载之后用户才能使用

windows下的程序不能直接在Linux中运行和安装

服务器注意事项

不要再服务器访问高峰时运行高负载命令

远程配置防火墙时不要把自己踢出服务器

1.文件处理命令

1.1目录处理命令

ls

ls 出来 如果是 - 开头代表是文件

如果是 d 代表是目录 是 l 代表是个软链接

r读 w写 x执行

rw- u 所有者 user

r– g 所属组 group

r– o 其他人 other

mkdir

mkdir 创建目录

/bin/mkdir

所有用户

mkdir -p [目录名] -p递归创建

cd

cd 切换目录

shell内置命令

所有用户

cd .. (这里必须要空格)切换上级目录

pwd

显示当前目录

print working directory

/bin/pwd

所有用户

rmdir

romove empty directories

删除空目录

rmdir [目录名]

cp

copy 复制

/bin/cp

cp /etc/grub.conf /tmp 将第一个复制到第二个目录下面 文件

cp -r 复制目录

cp 可以复制多个文件

cp -p 可以保存复制文件的属性 如创建时间等
